使用 systemd 创建定时任务

发表于 2025-06-15 | 分类于 OS

systemd timer 原理

在 Linux 中，使用 systemd 创建定时任务（即替代传统的 crontab）主要是通过两个单元（unit）文件来实现：

  • Service 单元（.service）：

    定义要执行的服务或命令

  • Timer 单元（.timer）：

    定义何时执行该任务

原理：systemd 定期检查每个 .timer单元中定义的规则，以确定是否触发相应的定时器，一旦定时器触发，systemd 将启动与该定时器关联的.service单元，从而执行预定的任务。
